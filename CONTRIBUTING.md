# 投稿指南

---

## 先说最简单的

如果你只是想说点什么，**开一个 Issue 就可以**。

不用 Fork，不用会 Git，也不用想太多格式。

点这里：<https://github.com/Wynphor/say-something/issues/new/choose>

选一个模板，写下内容，提交。

这样就已经被看见了。

---

## 💬 在回音壁发帖

回音壁是流动的评论区。

适合那些刚想到、想找人听见、也许会有人回应的话。

### 步骤

1. 点击「新建 Issue」
2. 选择一个接近的模板
3. 写下内容，提交

评论区可以自由留言。

如果一篇 Issue 被维护者标记为「精选」，它会自动搬到 `wall/`，并在原 Issue 下面留下链接。

---

## 📖 自己贴到壁报墙

如果你熟悉 GitHub，也可以直接往 `wall/` 投稿。

壁报墙是安静的角落，适合写得比较完整、想长期留下来的文字。

### 步骤

1. Fork 本仓库
2. 在 `wall/` 对应目录下新建 `.md` 文件
3. 文件命名：`化名-日期.md`（例：阿鱼-20260603.md）
4. 提交 PR
5. 通过校验后，系统会自动合并

### 文件模板

```markdown
# 标题（可选）

正文。想写什么写什么。

---
— 你的化名
```

### 目录说明

| 目录 | 适合放什么 |
|:---:|:---|
| `whisper/` | 心事、碎碎念、深夜的胡思乱想 |
| `story/` | 你的故事、你听来的故事、一段回忆 |
| `unsaid/` | 一直想说但没说出口的话 |
| `poetry/` | 诗、散文、文艺创作 |
| `daily/` | 今天的天气、路上的猫、吃到好吃的东西 |
| `gratitude/` | 想感谢的人、温暖的小事 |
| `archive/` | 转载别人的文字（请注明出处） |

---

## 投稿之后

### 如果你发的是 Issue

- 系统会自动送上一个 ❤️。
- 其他人可以回复，也可以只是点 reaction。
- 如果维护者觉得适合长期留下，会加上「精选」标签。
- 被精选后，内容会自动进入 `wall/`，原 Issue 会收到上墙链接。

### 如果你提的是 PR

- 只要改动都在 `wall/` 下，并且文件名和分类没问题，PR 会自动合并。
- 如果校验失败，GitHub 会在检查结果里说明原因。
- 上墙后 PR 会被标记为「壁报墙」和「已上墙」。

---

## 两者的区别

| | 📖 壁报墙 | 💬 回音壁 |
|:---:|:---|:---|
| 形式 | 仓库文件（PR） | Issue，精选后也可上墙 |
| 互动 | 可以评论 PR | 自由评论 |
| 节奏 | 安静、沉淀 | 即时、流动 |
| 适合 | 熟悉 GitHub 的投稿 | 大多数人，直接说就好 |

---

## 注意事项

- 自创和转载都可以，转载请注明来源
- 内容自由，没有字数要求
- 匿名或化名均可
